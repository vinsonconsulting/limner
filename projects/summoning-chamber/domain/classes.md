# Summoning Chamber — Class Domain Specifications

## Overview

Summoning Chamber features 11 distinct Classes, each representing a different agent role with unique furniture arrangements, equipment themes, and environmental characteristics. Every Class has furniture-themed environments that combine with Land backdrops to create complete master scenes.

**Purpose**: This document provides complete Class specifications for asset generation. Reference this when generating Class furniture overlays, Class icons, Class crests, or any asset tied to a specific Class's functional identity.

## Furniture Overlay Generation Guidelines

### Standard Workflow

1. **Select Class** from specifications above
2. **Review furniture items list** — understand complete environment
3. **Generate at 1024×1024 square** (more compliant than wide format)
4. **Use chromakey green background** for transparency keying
5. **Prompt structure**:
   ```
   SOLID BRIGHT GREEN BACKGROUND, collection of medieval [CLASS] furniture items,
   [furniture list from spec], isolated objects on green,
   NO WALLS NO ROOM, dark weathered materials, worn medieval furniture,
   grim fantasy aesthetic
   ```
6. **Crop Hillsfar text panel** (bottom-up scan, lower 60% only)
7. **Multi-tier keying**: Green chromakey → teal/cyan/black removal → wall color-range (if needed)
8. **Normalize to 480×270** via vga_normalize.py (hybrid stream, no dithering)
9. **Compose onto transparent canvas** (480×270)
10. **Nearest-neighbor upscale 4×** to 1920×1080

**Known Issues**:
- Hillsfar furniture style partially ignores chromakey green — fights toward full room scenes
- Square 1024×1024 format more compliant than wide 1820×1024
- "collection of items" prompt framing helps prevent room scene generation
- Text panel appears in EVERY generation — must crop before processing
- Wall removal can destroy gray furniture — use conservative color-range only

**Reference Workflow**: `core/workflows/furniture_overlay_generation.md`

### Class-Specific Prompt Modifiers

**For Tool-Heavy Classes** (Hammerer, Craftsman, Warden):
- Emphasize `organized tool arrangement`
- Specify `wall-mounted tool racks`
- Include `work surface with tools visible`

**For Knowledge Classes** (Scryer, Magister, Counselor):
- Emphasize `books, scrolls, texts`
- Specify `reading or study lighting`
- Include `organized knowledge storage`

**For Social Classes** (Diplomat, Herald, Merchant, Bard):
- Emphasize `welcoming arrangement`
- Specify `multiple seating or audience positions`
- Include `communication or performance focus`

**For Coordination Classes** (Seneschal):
- Emphasize `organizational systems`
- Specify `multi-project visibility`
- Include `central command position`

## Master Scene Composition

**Current Approach** (as of Session 10): Pre-composed master scenes replace layered composition to solve scaling mismatch issues.

### Letterboxing Technique

1. **Generate at 256×256 square** with 3 reference images:
   - Land backdrop (environment style)
   - Character sprite (denizen appearance)
   - Class overlay (furniture composition guide)

2. **Letterbox onto 480×270 canvas**:
   - Dark background RGB(20,18,15) — Summoning Chamber base tone
   - Scale square to 270×270 (nearest-neighbor)
   - Center horizontally (105px left offset)

3. **Upscale 4× to 1920×1080** (nearest-neighbor)

4. **Bonus**: Letterbox side areas (420px × 1080px each) provide natural UI zones for icons, status, inventory

**Reference Workflow**: `core/generation/letterboxing.md`

### Quality Gates for Master Scenes

- Character scale natural relative to furniture
- Land atmosphere preserved (palette, mood, lighting)
- Class environment correctly represented (right furniture types)
- VGA compliance (hard edges, atmospheric dithering only, no UI artifacts)
- Character positioned naturally on floor plane
- Furniture integrated at appropriate depths (not floating)
- Dark palette maintained throughout
- No text panels or decorative borders from generation
